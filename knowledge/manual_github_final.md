# Manual de Publicação Soberana: GitHub Pages (Sem Erros) 🚀🛡️

Se você está tendo erros técnicos (como erro de terminal ou 'mcp'), este guia foi feito para você. Vamos usar o método **Visual (Arrastar e Soltar)**, que não exige comandos e é 100% seguro.

## 1. Preparação (O que você precisa)

Certifique-se de que os arquivos `index.html`, `script.js` e `style.css` e a pasta `knowledge` estão prontos na sua pasta local: `c:\Users\User\.gemini\projetos\escritor-blog-master`.

## 2. Passo a Passo no GitHub (O Caminho Simples)

### Passo 01: Entrar no GitHub

1. Vá para [github.com](https://github.com) e faça login.
2. No canto superior direito, clique no botão **"+"** e escolha **"New repository"** (Novo Repositório).

### Passo 02: Criar a "Casa" do Projeto

1. Em **Repository name**, escolha um nome (ex: `meu-blog-master`).
2. Deixe como **Public** (Público) para que o Google Discover consiga ler.
3. Clique no botão verde abaixo: **"Create repository"**.

### Passo 03: Subir os Arquivos (Sem Comandos!) 🖱️

1. Na tela que abriu, procure pelo link azul: **"uploading an existing file"** (carregar um arquivo existente). Clique nele.
2. Abra a pasta do seu projeto no computador.
3. Selecione TODOS os arquivos e pastas (index, script, style e a pasta knowledge).
4. **Arraste e solte** tudo para dentro do quadrado cinza no site do GitHub.
5. Espere a barra verde terminar de carregar.
6. Lá embaixo, clique no botão verde: **"Commit changes"**.

### Passo 04: Colocar "No Ar" (Ativar o Link) 🌐

1. No menu superior desse repositório, clique em **"Settings"** (engrenagem).
2. Na barra lateral esquerda, clique em **"Pages"**.
3. Em "Branch", onde diz "None", mude para **"main"**.
4. Clique no botão **"Save"**.

## 3. Método 2: Usando o GitHub Desktop (O seu caso!) 🖥️

Como você já tem o aplicativo instalado, o processo é ainda mais fluido. Siga estes passos:

### Passo 01: Adicionar a Pasta

1. Abra o **GitHub Desktop**.
2. Vá no menu `File` -> `Add Local Repository` (Adicionar Repositório Local).
3. Clique em `Choose...` e selecione a pasta: `c:\Users\User\.gemini\projetos\escritor-blog-master`.
4. Se ele disser que "não é um repositório", clique no link azul que aparecer: **"Create a repository here"**.

### Passo 02: Publicar na Nuvem

1. No topo, você verá um botão azul escrito **"Publish repository"**. Clique nele.
2. Desmarque a opção "Keep this code private" (para o Discover funcionar mais fácil depois).
3. Clique em **"Publish"**.

### Passo 03: O "Commit" (Salvar Alterações)

1. Sempre que você mudar algo no Editor, o GitHub Desktop vai mostrar os arquivos na cor verde à esquerda.
2. Digite um nome no campo "Summary" (ex: "Atualização Blog").
3. Clique no botão azul embaixo: **"Commit to main"**.
4. Depois, clique lá no topo em **"Push origin"**. Isso envia tudo para a internet.

## 4. Passo Final (Importante!) 🌐

Mesmo usando o Desktop, você ainda precisa **"Ligar a Tomada"** do site:

1. Vá no site do GitHub no seu repositório.
2. Clique em **Settings** -> **Pages**.
3. Em "Branch", mude de "None" para **"main"**.
4. Clique em **Save**.

Dê 1 minuto e pronto: seu link estará brilhando no topo da página!
